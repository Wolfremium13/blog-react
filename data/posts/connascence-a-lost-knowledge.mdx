## ¬øQu√© es la Connascencia?

En ingenier√≠a de software, dos componentes son connascentes si un cambio en uno
requiere una modificaci√≥n en el otro para mantener el funcionamiento general del

### Fuerza de la Connascencia

Una forma de connascencia se considera m√°s fuerte si es m√°s probable que
requiera cambios compensatorios en los elementos connascentes. Cuanto m√°s fuerte
es la forma de connascencia, m√°s dif√≠cil y costoso es cambiar los elementos en
la relaci√≥n.

### Grado de Connascencia

La aceptabilidad de la connascencia est√° relacionada con el grado de su
ocurrencia. Un grado limitado de connascencia puede ser aceptable, pero un grado
grande puede ser inaceptable. Por ejemplo, una funci√≥n que toma dos argumentos
es generalmente aceptable, pero una que toma diez no lo es.

### Localidad de la Connascencia

La localidad es importante al analizar la connascencia. Formas m√°s fuertes de
connascencia son aceptables si los elementos involucrados est√°n estrechamente
relacionados. Por ejemplo, muchos lenguajes utilizan argumentos posicionales al
llamar a funciones o m√©todos, lo cual es aceptable debido a la cercan√≠a del
llamador y el llamado.

### Connascencias Est√°ticas

Son aquellas que se pueden encontrar examinando visualmente el c√≥digo.

#### Connascencia de Nombre (CoN)

Ocurre cuando m√∫ltiples componentes deben acordar el nombre de una entidad. Si
cambia el nombre de un m√©todo, los llamadores de ese m√©todo deben cambiar para
usar el nuevo nombre.

```python

    return a + b

# Despu√©s del cambio
def calcular_total(a, b):
    return a + b

# Los llamadores deben cambiar a usar el nuevo nombre

```

#### Connascencia de Tipo (CoT)

Se da cuando m√∫ltiples componentes deben acordar el tipo de una entidad. Si un
m√©todo cambia el tipo de su argumento, los llamadores deben pasar un argumento
diferente.

```python

    print(num * 2)

# Despu√©s del cambio
def procesar_numero(num: str):
    print(int(num) * 2)

# Los llamadores deben pasar un string en lugar de un entero

```

#### Connascencia de Significado (CoM) o Connascencia de Convenci√≥n (CoC)

Sucede cuando m√∫ltiples componentes deben acordar el significado de valores
particulares.

```python

    # 0 para inactivo, 1 para activo
    return "Activo" if codigo == 1 else "Inactivo"

# Despu√©s del cambio

    # 'A' para activo, 'I' para inactivo
    return "Activo" if codigo == 'A' else "Inactivo"

# Los llamadores deben usar 'A' o 'I' en lugar de 0 o 1

```

#### Connascencia de Posici√≥n (CoP)

Ocurre cuando m√∫ltiples componentes deben acordar el orden de los valores.

```python

```

Se presenta cuando m√∫ltiples componentes deben acordar un algoritmo particular.

#### Connascencia de Valores (CoV)

Cuando varios valores deben cambiar juntos.


## Reduciendo la Connascencia

```python

    print(f"Usuario: {nombre} {apellido}")

# Despu√©s del cambio

    print(f"Usuario: {nombre} {apellido}")
```
